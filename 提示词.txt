====================
二、你必须实现的功能
====================

你需要实现一个完整的量化计算模块，用于将实例分割 mask 转换为工程可用的几何量化结果。

核心原则：
- 不允许只用 bbox 粗略测量裂缝；
- 对 Crack 类必须使用“骨架长度 + 局部宽度估计”的方式；
- 对面状病害优先使用 mask/contour 计算面积、周长、方向、外接矩形等；
- 物理单位换算必须是“可选”的：如果没有提供比例尺，则只输出像素单位；如果提供比例尺，则同时输出物理单位；
- 绝对不能在没有比例尺的情况下伪造 mm、cm、m 单位结果。

你需要实现至少以下能力：

1. 新建量化模块文件
建议新建：
- quantification.py
或
- bridge_quantify.py
名称你自行判断，但必须清晰。

2. 针对单个实例 mask 计算基础几何量
至少包括：
- area_px：像素面积
- perimeter_px：像素周长
- bbox_xyxy / bbox_xywh
- centroid
- orientation_deg
- major_axis_px
- minor_axis_px
- compactness
- extent 或 area_ratio_in_bbox

3. 对 Crack 类实现精细量化
Crack 类必须额外输出：
- length_px
- mean_width_px
- max_width_px
- p95_width_px
- skeleton_pixel_count
- branch_count（如果实现难度较大，可给近似实现，但不能乱写）
- 说明字段 measurement_method="skeleton_distance_transform"

Crack 的长度计算要求：
- 不能用 bbox 长边替代；
- 必须基于 skeleton / thinning / medial-axis 思路；
- 尽量按 8 邻接图长度求和，水平垂直按 1、对角按 sqrt(2)；
- 如果环境中无 skimage/scipy，提供不崩溃的 fallback；
- 优先使用现有依赖，避免引入沉重依赖；
- 如果必须新增依赖，必须非常轻量，并在最终说明中写明。

Crack 的宽度计算要求：
- 不能直接用 bbox 短边；
- 必须基于 mask 的局部宽度估计；
- 优先使用 distance transform；
- 输出 mean_width_px / max_width_px / p95_width_px；
- max_width_px 尽量避免被单点噪声污染，必要时可同时保留 raw max 和稳健 max，但至少要有 p95_width_px。

4. 对非 Crack 类实现面状病害量化
对 Breakage、ReinForcement、Comb、Seepage、Hole 等非 Crack 类，至少输出：
- area_px
- perimeter_px
- major_axis_px
- minor_axis_px
- min_area_rect
- orientation_deg
- compactness
- extent 或 bbox_fill_ratio

5. 可选物理尺度换算
量化模块必须支持两种模式：
- 未提供比例尺：仅输出 px 单位
- 提供比例尺：输出 px + physical 单位

最少支持：
- --mm-per-pixel 单值输入
即：
length_mm = length_px * mm_per_pixel
width_mm = width_px * mm_per_pixel
area_mm2 = area_px * mm_per_pixel^2

你还需要为后续扩展预留接口：
- 预留 scale_config / image_scale_map / per-image scale 的入口
- 但当前不必强制实现复杂标定
- 可以在代码中写清楚：未来可扩展 homography / calibration / per-image scale

6. mask 预处理
在测量前对 mask 做适度净化，避免毛刺和噪声影响结果：
- 二值化
- 去除极小连通域
- 可选开闭运算
- 保证不会把细裂缝全部抹掉
- Crack 和非 Crack 的净化策略可以不同

7. 输出结构化结果
每张图像除原始可视化图外，还要输出结构化量化结果，建议：
- 一个 image-level JSON
- 一个全局汇总 CSV

建议输出目录：
- test_results/quant_json/
- test_results/quant_csv/
- test_results/quant_vis/

每个实例至少包含：
- image_name
- instance_id
- class_id
- class_name
- score
- bbox
- mask_area_px
- geometry metrics
- crack metrics（若为 Crack）
- physical metrics（若有比例尺）
- measurement_confidence
- warnings

8. 可视化增强
在原有检测图基础上，新增量化可视化版本，至少展示：
- 类别名
- score
- area_px
- Crack 的 length/width（若适用）
- 若提供比例尺，显示 mm/mm²
要求：
- 不要覆盖破坏原始 detected 图
- 新增一个量化可视化输出版本
- 文本布局尽量清晰，不要严重遮挡目标

9. measurement_confidence
你需要给每个实例一个简单、可解释的 measurement_confidence（0~1）。
可以综合考虑：
- detection score
- mask 面积是否过小
- 是否触边
- mask 是否过于破碎
- Crack skeleton 是否稳定
- 是否成功完成宽度估计

要求：
- 逻辑清楚
- 不要伪装成复杂 AI
- 采用规则法即可
- 如果无法可靠测量，confidence 降低并给 warning

10. 异常处理
必须处理：
- 空 mask
- 全零 mask
- 极小目标
- 触边目标
- 多连通域
- 读取失败
- 比例尺缺失
- 目录不存在
- 单张图与批量图两种模式
要求：
- 不崩溃
- 输出 warning
- 跳过非法实例但不影响整图处理
